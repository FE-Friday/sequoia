# UNION 操作符

MySQL UNION 操作符用于连接两个以上的 SELECT 语句的结果组合到一个结果集合中。多个 SELECT 语句会删除重复的数据。

# 语法

SELECT expression1, expression2, ... expression_n
FROM tables
[WHERE conditions]
UNION [ALL | DISTINCT]
SELECT expression1, expression2, ... expression_n
FROM tables
[WHERE conditions];

TIP

  • expression1, expression2, ... expression_n: 要检索的列。
  • tables: 要检索的数据表。
  • WHERE conditions: 可选， 检索条件。
  • DISTINCT: 可选，删除结果集中重复的数据。默认情况下 UNION 操作符已经删除了重复数据，所以 DISTINCT 修饰符对结果没啥影响。
  • ALL: 可选，返回所有结果集，包含重复数据。

# UNION 实例

下面的 SQL 语句从 "Websites" 和 "apps" 表中选取所有不同的country（只有不同的值）：

mysql> select country from websites union select country from apps order by country;
+---------+
| country |
+---------+
| CN      |
| USA     |
+---------+
2 rows in set (0.00 sec)

TIP

注意：UNION 不能用于列出两个表中所有的country。 如果一些网站和APP来自同一个国家，每个国家只会列出一次。 UNION 只会选取不同的值。 请使用 UNION ALL 来选取重复的值！

# 连接「多表联查」

我们已经学会了如何在一张表中读取数据，这是相对简单的。

真正的应用中经常需要从多个表中读取数据。

这里介绍如何使用 MySQL 的 JOIN 在两个或多个表中查询数据。

你可以在 SELECT, UPDATE 和 DELETE 语句中使用 Mysql 的 JOIN 来联合多表查询。

JOIN 按照功能大致分为如下三类：

TIP

  • INNER JOIN（内连接,或等值连接）：获取两个表中字段匹配关系的记录。
  • LEFT JOIN（左连接）：获取左表所有记录，即使右表没有对应匹配的记录。
  • RIGHT JOIN（右连接）： 与 LEFT JOIN 相反，用于获取右表所有记录，即使左表没有对应匹配的记录。

# inner join 实例

使用 inner jion 连接两张表来读取 websites 表中所有的 id 字段在 access_log 表对应的 count 字段值：

mysql> select a.id, a.name, b.count from websites a inner join access_log b on a.id = b.site_id;
